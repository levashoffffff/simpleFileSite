# Простой новостной сайт на PHP с использованием файлов

Простой новостной сайт, который использует локальную файловую систему в качестве базы данных. Сайт создан с использованием PHP и Bootstrap.

## Структура проекта

- `index.php`: Основная точка входа. Отображает список новостных статей.
- `admin/`: Содержит панель администратора для управления контентом.
- `config/`: Хранит конфигурационные файлы, такие как учетные данные для доступа (в данном случае, данные для входа в панель администратора).
- `css/`: Содержит таблицы стилей, включая Bootstrap.
- `data/`: Выполняет роль базы данных. Каждый `.txt` файл в этой директории представляет собой новостную статью, сохраненную в виде сериализованного массива PHP.
- `js/`: Содержит JavaScript файлы.
- `modules/`: Содержит модули PHP для определенных функций.
  - `readnews.php`: Читает все статьи из директории `data`.
  - `news.php`: Отображает одну новостную статью.
- `templates/`: Содержит переиспользуемые части HTML-структуры.
  - `header.php`: Заголовок страницы.
  - `menu.php`: Навигационное меню.
  - `login.php`: Страница входа в панель администратора.

## Как это работает

Сайт читает `.txt` файлы из директории `/data`. Каждый файл содержит сериализованный PHP массив с ключами 'id', 'title' и 'text'. Главная страница (`index.php`) отображает список всех новостей, а конкретную статью можно просмотреть, передав ее ID в URL (например, `/?act=news&id=1`).

## Известные проблемы

- **Сломанная аутентификация:** Механизм входа в систему в настоящее время сломан. Форма входа неправильно устанавливает сессию, что препятствует доступу к панели администратора.
- **Небезопасная генерация ID:** ID для новых статей генерируются простым подсчетом количества файлов в директории `data`. Это ненадежный метод, который может привести к перезаписи существующих статей, если одна из них будет удалена.
- **Отсутствие функции удаления:** В панели администратора нет возможности удалять статьи.
